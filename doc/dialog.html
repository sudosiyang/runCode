dialog

概述

特殊UI框架能够适应内容变化，甚至连外部程序动态插入的内容它仍然能自适应，因此你不必去考虑消息内容尺寸使用它。它的消息容器甚至能够根据宽度让文本居中或居左对齐——这一切全是XHTML+CSS原生实现

作用
dialog插件
参数:
名称 类型 属性 默认 描述
title String <任意>
 null 标题内容
content String/Element <任意>
 如果传入的是HTMLElement类型，
如果是隐藏元素会给其设置display:block以显示该元素，
其他属性与绑定的事件都会完整保留
ok Callback <任意>
 确定按钮回调函数。函数如果返回false将阻止对话框关闭；
函数this指针指向内部api；
如果传入true表示只显示有关闭功能的按钮
okVal String <任意>
 确定 "确定按钮"文字
cancel Callback <任意>
 取消按钮回调函数。
函数如果返回false将阻止对话框关闭；
函数this指针指向内部api；
如果传入true表示只显示有关闭功能的按钮
对话框标题栏的关闭按钮其实就是取消按钮，
只不过视觉不同罢了，点击同样触发cancel事件
cancelVal String <任意>
 取消 "取消按钮"文字
copy Callback <任意>
 复制按钮，复制文字,成功时回调
copy_target String <任意>
 .textarea 复制目标
width Number <任意>
 auto 设置消息内容宽度，可以带单位。一般不需要设置此，对话框框架会自己适应内容。
如果设置为百分值单位，将会以根据浏览器可视范围作为基准，
此时如果浏览器窗口大小被改变其也会进行相应的调整
height Number <任意>
 auto 设置消息内容高度，可以带单位。
不建议设置此，而应该让内容自己撑开高度。
如果设置为百分值单位，将会以根据浏览器可视范围作为基准，
此时如果浏览器窗口大小被改变其也会进行相应的调整
drag Boolen <任意>
 false 是否允许用户拖动位置
time Number <任意>
 设置对话框显示时间。以毫秒为单位
版本:

示范:
// $PATH为索引路径
var Pop=require("$PATH/lib/util/dialog/dialog");
Pop.dialog({
    title:"这是一个测试",
    content:$(".con"),
    ok:function(){
  		alert();
  	},
    drag:true,
    copy:function(){
    	this.tip();
    },
    copy_target:".con",
  })
